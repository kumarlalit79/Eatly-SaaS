Eatly MVP Plan (Phase 1 SaaS)

1. Product Overview
   1.1 One-Liner Vision

Eatly is an AI-powered SaaS that scans any restaurant menu and instantly tells you what each dish is, whether it is veg/non-veg, how healthy it is, and what you should order.

1.2 User Pain Point Solved

When people go to a restaurant (in India or abroad), they often face confusion because:

They don’t understand unfamiliar dish names, especially regional or international dishes. They don’t know what ingredients are inside the food. They don’t know whether the dish is vegetarian or non-vegetarian. They also don’t know whether the dish is healthy, oily, fried, sugary, or heavy. Because of this, they waste time searching on Google, asking the waiter, or ordering randomly and regretting later.

Eatly solves this by giving instant menu clarity in one scan, so users can confidently choose the best meal without confusion.

2. Finalized MVP Features (Must-Haves for Phase 1)

These are the locked Phase 1 features based on MoSCoW prioritization. The goal is to ship fast but still deliver strong value.

2.1 Menu Scan + OCR Extraction

The user can upload or scan a menu photo, and Eatly will extract all dish names and menu text using OCR. This is the base feature because the product cannot work without accurate menu reading.

2.2 Dish Explanation in Simple English

Once the menu is extracted, Eatly will explain what each dish is in simple language. It will describe what type of dish it is, what it generally contains, and what it tastes like or how it is cooked.

This is the main feature that solves the “I don’t know what this dish means” problem.

2.3 Veg / Non-Veg Classification

Eatly will classify every dish into categories like vegetarian or non-vegetarian. If possible, it will also detect egg, seafood, or dairy-based dishes. This feature is extremely useful for Indian users, especially when exploring different states or foreign cuisines.

2.4 Health Score + Reason

Eatly will show a clear health rating such as:

Healthy

Moderate

Avoid

Along with this, it will also give a short reason like “deep fried,” “high sugar,” “high butter/cream,” or “grilled and high protein.”

This makes Eatly different from a simple translator app, because it gives real decision-making help.

2.5 Top Recommended Items from the Menu

Eatly will highlight the top recommended dishes based on health score. It will show something like “Top 5 healthiest items in this menu.” This makes the product feel like a personal food assistant instead of just an information tool.

This feature increases user satisfaction because it saves time and directly tells them what to order.

3. Detailed User Journey (Landing Page → SaaS Experience)

This is the full flow for a user who discovers Eatly and starts using it.

3.1 Landing Page Entry

A user lands on Eatly’s landing page through Instagram, Google, a referral, or a social media post. They are usually hungry and want a quick answer, so they don’t have patience for complicated explanations.

They immediately see a clear message like:

“Scan any restaurant menu. Eatly tells you what to eat, what’s healthy, and what to avoid.”

This builds instant interest.

3.2 User Scrolls and Builds Trust

The user scrolls the landing page and sees a demo section that visually shows:

Menu image → AI analysis → results with labels.

This reduces doubt and makes the product feel real. They understand exactly what they will get.

Then they see simple benefits such as:

Understand any dish instantly

Find veg/non-veg easily

Avoid oily or unhealthy meals

Get the best menu recommendations

Now the user feels that Eatly is useful.

3.3 User Clicks “Try Free”

The user clicks the main CTA button, such as “Try Eatly Free” or “Scan a Menu Now.” This takes them into the SaaS web app.

3.4 Signup / Login

User is asked to sign up quickly using Google or email. Since it is an MVP, it should be fast and minimal, with no unnecessary onboarding forms.

The goal is to get the user inside the product in under 15 seconds.

3.5 Dashboard Welcome Screen

After login, the user lands on a simple dashboard with one clear action:

Upload Menu Photo

There should be no distractions. The product must guide them directly to the main action.

3.6 Upload Menu Screen

User uploads a photo of the menu.

They see the preview of the image and click:

Scan Menu

This is the key action stage where the user starts feeling excited because they expect an instant solution.

3.7 AI Scanning and Processing

Eatly shows a clean loading UI with steps like:

Extracting text from menu

Identifying dishes

Detecting veg/non-veg

Scoring health rating

Finding best recommendations

Even if it is just animation, it makes the product feel premium.

3.8 Results Page (Magic Moment)

After scanning, user sees the full menu in structured format.

Each dish shows:

Dish name

Veg/Non-veg badge

Health badge (Healthy/Moderate/Avoid)

One-line explanation

This is the “wow moment” because now the user finally understands everything in seconds.

3.9 Dish Details Page (Optional Click)

If the user clicks any dish, they see a detailed view that explains:

what it is

ingredients (estimated)

cooking style

why it is healthy/unhealthy

This builds trust and gives more clarity.

3.10 Top Picks Section

On the results page, user sees a section like:

Top 5 Healthy Picks from This Menu

This is the most actionable part of the product because it tells the user what they should order immediately.

3.11 Scan History (Retention)

After the scan is completed, the menu is saved in their history. Next time they open Eatly, they can access their previous scans without uploading again.

This increases repeat usage and retention.

3.12 Paywall / Upgrade Trigger

Once the user uses their free scans limit, Eatly shows a clean upgrade screen:

“You’ve reached your free scan limit. Upgrade to Eatly Pro for unlimited menu scans.”

This works because the user already received value, so they are more likely to pay.

4. Monetization Plan (Free vs Pro)

Eatly should follow a simple SaaS subscription model. Since this is a consumer-friendly product, pricing should be affordable.

4.1 Free Plan (For Virality + Growth)

The Free plan should be designed to give real value but still create a reason to upgrade.

Free Plan Features

User can:

Upload and scan menu (limited scans)

Get dish explanations

Get veg/non-veg classification

Get health score (basic)

Get top recommendations (limited)

Free Plan Limits

You can set limits like:

3 scans total free
or

5 scans per month free

This ensures people experience the product but still hit the paywall quickly.

The free plan exists mainly to spread the product and create trust.

4.2 Pro Plan (Paid Subscription Plan)

The Pro plan should feel like the “real Eatly experience.”

Pro Plan Features

User gets:

Unlimited menu scans

Faster processing speed

Full dish explanation + detailed breakdown

Accurate veg/non-veg + egg + seafood detection

Full health scoring system

Top 5 healthiest items in every menu

Menu scan history storage (long-term)

This plan should feel like a “food assistant subscription.”

4.3 Recommended Pricing (Simple)

Since your audience includes Indian users, pricing should be friendly.

Best SaaS pricing options:

Monthly plan: ₹199 to ₹299/month

Yearly plan: ₹999 to ₹1499/year

Yearly should feel like a steal.

This also improves cashflow early.

5. MVP Success Metrics (What to Track After Launch)

Once MVP is launched, you should track these metrics to know if Eatly is working.

5.1 Activation Metric

How many users upload a menu within the first 5 minutes after signup.

This tells if your product is clear and engaging.

5.2 Retention Metric

How many users return and scan again within 7 days.

This tells if the product becomes a habit.

5.3 Conversion Metric

How many users upgrade after hitting scan limit.

This tells if users value Eatly enough to pay.

Final Summary (The Complete Phase 1 MVP)

Eatly Phase 1 MVP is a SaaS that delivers one powerful promise:

The user uploads a restaurant menu, and Eatly instantly explains dishes, marks veg/non-veg, scores healthiness, and recommends the best healthy dishes from the menu. Users can use it free for a limited number of scans, and upgrade to Pro for unlimited scans and full menu analysis.

This MVP is fast to build, easy to market, and provides immediate real-world value, which makes it perfect for launching quickly and validating demand.
